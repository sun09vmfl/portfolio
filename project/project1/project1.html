Cording Convention

  • 1. 파일 및 폴더명은 소문자로 작성
  • 2. 메인페이지명은 index.html, 서브페이지명은 project로 작성
  • 3. image는 각각의 project 폴더 하위에 새로 생성
  • 1. reset 파일은 minireset.min.css 파일을 기본 사용하되, 필요한 것과 필요없는 것을 첨삭.
  • 2. 폰트는 웹폰트와 fonts.css 파일 사용.
  • 3. style.css와 js 파일 생성시, html과 동일하게 생성.
  • 1. html, css의 id 및 class 값은 기본적으로 소문자로 작성하되 두 단어 이상 조합 시 기본 '-' 하이픈 작성. js는 카멜표기법과 '_' 언더바 작성.
  • 2. querySelectorAll 사용 시, 변수명은 s를 붙여 인지에 용이하게 함.
  • 3. html의 구조는 header, bodt, footer로 구조를 분명하게 함.
  • 1. 들여쓰기는 기본 tab을 사용. (tab 사이즈는 2로 통일)
  • 2. 키워드, 연산자와 다른 코드 사이에는 가독성을 위해 공백 작성
  • 3. 변수 선언 시 const, let 사용.
  • 4. 괄호의 위치는 K&R 스타일로 작성.
  • 5. 기본적인 작성 방식은 vscode 내 prettier 작성 방식을 따름.
