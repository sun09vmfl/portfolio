Convention

Cording Convention

  • 1. 파일 및 폴더명은 소문자로 작성
  • 2. 메인페이지명은 index.html, css는 main.css, js 파일은 js.js로 적은 파일 양이므로 직관적이게 표시.
  • 3. image 파일은 일관성 있는 이름으로 재변경.
  • 1. reset 파일은 minireset.min.css 파일을 기본 사용하되, 필요한 것과 필요없는 것을 첨삭.
  • 2. 폰트는 웹폰트 사용.
  • 3. style.css와 js 파일 생성시, html과 동일하게 생성.
  • 1. html, css의 id 및 class 값은 기본적으로 한 단어 위주로 사용. 늘어날 경우 타 라이브러리 클래스 명처럼 '-' 하이픈 사용
  • 2. querySelectorAll 사용 시, 변수명은 s를 붙여 인지에 용이하게 함.
  • 3. html의 구조는 header, bodt, footer로 구조를 분명하게 함.
  • 1. 들여쓰기는 기본 tab을 사용. (tab 사이즈는 2로 통일)
  • 2. 키워드, 연산자와 다른 코드 사이에는 가독성을 위해 공백 작성
  • 3. 변수 선언 시 const, let 사용.
  • 4. 괄호의 위치는 K&R 스타일로 작성.
  • 5. 기본적인 작성 방식은 vscode 내 prettier 작성 방식을 따름.

Solution

화려함을 구현하기 위한 라이브러리 활용

1. 자바스크립트를 통해 많은 구현을 해 봤으므로, 이번에는 플러그인 라이브러리를 많이 사용해 보았다.

2. 글자 하나 하나를 span으로 묶어, aos와 transform으로 구현을 시도했으나, 스크롤 이벤트와 더불어 애니메이션의 충돌로 인해 지저분한 코드와 정갈한 모양이 나오지 않았다. 텍스트 애니메이션 jQuery (Splitting.js)를 통해 간편하고 본래 사이트와 똑같이 구현 가능.

3. 게이지바 애니메이션 구현, 로딩창과 맞추기 위해 시간을 맞춤.

Abosolute의 두 얼굴

1. 많은 사진, 화려한 요소들을 위치시키기 위해 필수적인 position: absolute. 웹 기준으로 사이트 제작을 했지만 다른 해상도의 브라우저에서 보면 다르게 보이는 경우가 잦음. 범위를 퍼센트, 혹은 vh, vw로 해결.

2. 기존 div box를 통해 구현한 것의 오차가 생겨서 가상 선택자를(before,after) 이용하여 마우스 오버 효과를 줌.

Animation의 늪

1. 마우스 스크롤에 따른 애니메이션을 부여 시 첫 사용하는 jQuery를 통하여 제작 if와 else문을 활용하여 마우스 휠 위치에 따른 애니메이션 부여

Review

혼자서 빠른 사이트 제작을 위해 새롭게 익힌 제이쿼리와 플러그인을 더욱 활용하여 좀 더 화려하고 인터랙티브 한 사이트를 만들고 싶어 도전하게 된 웹사이트입니다.

화려함과 더불어 많은 정보를 담고 있어 신뢰를 줄 수 있는 느낌을 구현하기 위해 노력했습니다.

제이쿼리는 물론이고 많은 플러그인을 사용하고, 찾아 보며 어떤 것들끼리는 충돌 없이 수월한 활용이 가능한지 알 수 있는 좋은 기회가 되었습니다.

화려하지만, 정신 없지는 않게.
웹 사이트 제작에 있어서 가장 중요한 것을 깨달았습니다. 화려해서 이목을 끌지만 안에 담긴 정보를 잘 전달할 수 있도록 하는 것이 웹사이트 제작에 있어서 가장 중요한 요소라고 생각합니다.
